2. Verarbeitete personenbezogene Daten

2.1 Benutzerkonto

Zur Nutzung von JobPipeline können Sie ein Konto erstellen. Dabei werden folgende Daten gespeichert:

2.2 Gespeicherte Stellenanzeigen

Wenn Sie Stellen auf dem Merkzettel speichern, werden diese Daten (Jobtitel, Unternehmen, URL, Ihre Notizen und Bewerbungsstatus) serverseitig Ihrem Konto zugeordnet gespeichert.

2.3 Karriere-Monitor (Watchlist)

Wenn Sie Unternehmens-Karriereseiten zur Beobachtung hinzufügen, werden gespeichert: URL der Karriereseite, Unternehmensname, Suchbegriffe, Prüfintervall sowie gefundene Stellenanzeigen. Diese Daten werden ausschließlich für den automatischen Abgleich verwendet.

2.4 Session-Cookie

Nach der Anmeldung wird ein HTTP-only Session-Cookie gesetzt, um Sie eingeloggt zu halten. Dieser Cookie enthält keine personenbezogenen Daten, sondern eine zufällige Session-ID. Er wird beim Abmelden oder nach Ablauf der Session gelöscht.

2.5 Server-Logs

Der Webserver (Caddy) und die Anwendung können technische Zugriffsdaten protokollieren (IP-Adresse, Zeitstempel, aufgerufene URL). Diese Daten dienen ausschließlich der Fehlerdiagnose und Sicherheit und werden nach spätestens 7–30 Tagen gelöscht.

3. Externe Dienste und Datenweitergabe

3.1 Adzuna API

Für die Jobsuche werden Suchanfragen (Jobtitel, Ort, Umkreis) an die API von Adzuna Ltd. übermittelt. Adzuna verarbeitet diese Daten gemäß ihrer Datenschutzerklärung. Es werden keine Kontodaten oder personenbezogene Nutzerinformationen an Adzuna weitergegeben.

3.2 Bundesagentur für Arbeit / Jobicy

Suchanfragen können ebenfalls an die API der Bundesagentur für Arbeit sowie Jobicy übermittelt werden. Auch hier werden nur Suchparameter (Jobtitel, Ort), keine Nutzerdaten übertragen.

3.3 Jira-Integration (optional)

Falls Sie die optionale Jira-Anbindung konfigurieren, werden Stellendaten von JobPipeline an Ihre eigene Jira-Instanz (Atlassian Cloud) übertragen. Diese Verbindung läuft über Ihren persönlichen API-Token. Atlassian ist dabei ein eigenständiger Verantwortlicher gemäß DSGVO.

3.4 Keine weiteren Drittanbieter

Es werden keine Tracking-Dienste, Werbenetzwerke, Analytics-Tools (Google Analytics o.Ä.) oder Social-Media-Plugins eingesetzt. Es findet keine Profilbildung statt.

4. Speicherdauer und Löschung

Personenbezogene Daten werden gelöscht, sobald der Zweck der Speicherung entfällt:

  • Kontodaten: Bei Löschung des Kontos durch den Nutzer oder durch den Administrator werden alle zugehörigen Daten (gespeicherte Stellen, Watchlist) unwiderruflich gelöscht.
  • Session-Cookie: Wird beim Abmelden oder nach Ablauf der Session gelöscht.
  • Server-Logs: Werden nach spätestens 30 Tagen automatisch gelöscht.
  • Backups: Automatische Datensicherungen werden nach 30 Tagen überschrieben.

  • Auskunftsrecht (Art. 15): Welche Daten wir über Sie verarbeiten.

  • Löschungsrecht (Art. 17): Löschung Ihres Kontos und aller Daten auf Anfrage oder durch Selbstlöschung.
  • Einschränkung der Verarbeitung (Art. 18): Auf Anfrage möglich.
  • Datenübertragbarkeit (Art. 20): Export aller Ihrer Daten als JSON über die Backup-Funktion im Admin-Bereich.

6. Beschwerderecht bei der Aufsichtsbehörde

Sie haben das Recht, sich bei einer Datenschutz-Aufsichtsbehörde zu beschweren. Die zuständige Behörde richtet sich nach dem Bundesland des Verantwortlichen. Eine Übersicht aller deutschen Aufsichtsbehörden finden Sie unter bfdi.bund.de.

7. Datensicherheit

Die Übertragung zwischen Browser und Server erfolgt ausschließlich verschlüsselt über HTTPS (TLS). Passwörter werden serverseitig mit bcrypt gehasht und niemals im Klartext gespeichert oder übertragen. Der Zugriff auf die Datenbank ist auf den Anwendungsserver beschränkt.
